# Stryker Cyberattack — CVE Research Report

## Incident Summary

On **March 11, 2026**, U.S.-based medical-technology giant **Stryker Corporation** suffered a destructive cyberattack that disrupted its global Microsoft environment, reportedly bricking/wiping tens of thousands of employee endpoints (figures reported between ~80,000 and ~200,000 devices). Responsibility was claimed by **Handala Hack Team**, an Iran-linked hacktivist group tied to Iran's Ministry of Intelligence and Security (MOIS); some analysts also link pre-positioning to **MuddyWater (Seedworm)**.

Key technical findings publicly reported:
- **No CVE/exploit was used as the initial vector.** Stryker and outside researchers (Sygnia, Coalition, glueckkanja, SecurityWeek) concluded that attackers used **infostealer-derived credentials** for an admin account, then abused **Microsoft Intune** (and Entra ID privileges) "as designed" to push remote-wipe commands. A non-propagating malicious file was found, but no ransomware/wiper malware on endpoints was needed.
- CISA issued an alert on March 18, 2026 urging hardening of endpoint-management systems.

Because the attack used legitimate identity/MDM features rather than software exploits, no CVE was the *root cause*. However, multiple security vendors, threat-intelligence reports, and weekly threat bulletins discuss the Stryker incident **alongside** specific CVEs — either as related Iranian TTPs, as MDM/identity-stack lessons-learned, or as concurrent active-exploitation alerts. Those CVEs are listed below.

## Important Caveats

1. **No CVE has been confirmed as the initial-access vector for the March 2026 Stryker breach.** Public reporting (Stryker, Sygnia, Coalition, SecurityWeek, glueckkanja) attributes the compromise to **stolen credentials via infostealer malware** abused through legitimate **Microsoft Intune / Entra ID** functionality — not a software vulnerability.
2. The CVEs above are primarily discussed in **post-incident lessons-learned articles**, **same-week threat bulletins**, or **broader Iranian-actor campaigns** that overlap with the Stryker attack timeframe. Treat them as *related context*, not as confirmed exploits in the Stryker attack itself.
3. Several articles dated March/April **2026** use forward-dated CVE identifiers (CVE-2026-xxxx). These reflect the incident's actual reporting timeframe.
